# **Face Attributes Retrieval Demo**

**Authors**: Romane NGOUAH, Khadija WAHHABI (P21) Course: Deep Learning (E4)



##### **Project Overview**

This project is a Visual Search Engine based on the CelebA dataset (containing over 200,000 images of celebrities). It allows users to search for images by selecting semantic attributes (e.g., "Smiling", "Eyeglasses", "Blond Hair") rather than using text keywords or image inputs.



The system uses a Deep Learning approach combining a ResNet-50 backbone for image feature extraction and a custom MLP (Multi-Layer Perceptron) to project textual attributes into a shared embedding space.



**Note**: The core focus of this project was the Deep Learning model implementation and embedding space analysis. The web interface code (Flask/HTML/CSS) was generated with the assistance of Google Gemini, based on our specific requirements, to provide a functional demo within the project timeline.



##### **Key Features**

* **Semantic Search:** Select from 1 to 40 physical attributes to filter images.
* **Dynamic Results:** Choose the number of images to retrieve (Top-K) via a slider (from 5 to 50 results).
* **Real-time Inference:** Fast retrieval using pre-computed embeddings and matrix operations (Euclidean distance).
* **Interactive Interface:** Clean web UI with responsive design, attribute highlighting, and dynamic feedback.
* **Hybrid Path System:** The application automatically detects the dataset location, supporting both a portable mode (inside project) and a local user mode.

##### **Installation \& Setup**

**1. Prerequisites**

Ensure you have Python 3.x installed. You will need to install the following dependencies via your terminal:



pip install flask torch torchvision numpy



**2. Dataset Configuration**

The application needs access to the CelebA dataset (the "img\_align\_celeba" folder containing the .jpg images). **Dataset Source:** https://mmlab.ie.cuhk.edu.hk/projects/CelebA.html



The app.py script automatically checks two locations in this priority order:



**Option A (Portable Mode):** Place the "img\_align\_celeba" folder directly inside the project root folder (next to app.py). This is recommended if you want to share the project folder with someone else.



**Option B (User Home Mode):** Keep the dataset in your user documents at the following path: C:/Users/YOUR\_NAME/FaceAttributes/img\_align\_celeba/



**3. Model Files**

Ensure the "model/" folder contains the specific files generated by the training notebook:



embeddings\_resnet50.pt

celeba\_attributes.pt

celeba\_image\_names.pt

mlp\_turbo.pth

resnet\_turbo.pth



Note: These files must match the architecture defined in the app.py script.

##### **Technical Details**

* **Image Encoder:** ResNet-50 (Pre-trained on ImageNet, used as a Feature Extractor).
* **Attribute Encoder (MLP):** A custom neural network transforming binary attribute vectors (size 40) into 300-dimensional embeddings.
* **Training Method:** Joint Training with Triplet Loss to enforce closeness between an image and its corresponding attributes in the vector space.
* **Retrieval:** The search computes the Euclidean Distance between the query embedding and all 200k+ database embeddings to find the nearest neighbors (Top-K).
